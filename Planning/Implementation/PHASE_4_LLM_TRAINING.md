# PHASE_4_LLM_TRAINING.md — Project ATAL
**Scope: Supervised fine-tuning (SFT) of the worker LLM `qwen3.5:0.8b` for the two-tier ATAL agentic architecture, using Unsloth Studio. Domain: industrial maintenance reasoning over Horizon & Zephyr.**

Task format: `- [ ] P4-NNN: {task} | REQ: REQ-LLM-NNN | Depends: P4-NNN`

> **Architecture recap.** MANAS supervisor (`qwen3.5:9b`) plans, reasons over the consolidated
> payload + RAG, and decides actions. The worker (`qwen3.5:0.8b`) runs 5 bounded, parallel
> text-transform sub-tasks defined in `apps/agents/graph/agents.py`:
> `work_order_drafter`, `sensor_window_summarizer`, `alarm_triager`, `citation_formatter`,
> `spare_strategist`. Each worker receives a fixed **system** prompt + a **user** prompt filled
> with ML-derived data (RUL, anomaly_score, bottleneck_score, cross-stage correlations, 24h sensor
> stats, spares) and returns a short text artifact the supervisor consumes.
>
> **This phase fine-tunes the 0.8b worker** so its outputs are consistent, on-format, and
> domain-grounded — reliable enough to offload work from the 9B supervisor. The **dataset is
> generated by the team**; this document is the training runbook + parameter spec.
>
> **Constraints (in force):** self-hosted only, no external API (REQ-SECURITY-005). Final artifact
> is **GGUF**, served by the existing `ollama` container behind `OLLAMA_SMALL_MODEL`
> (`keep_alive=30m`). No `openai`/`anthropic`/`langchain-openai`. **Never** vLLM (GGUF unsupported).

## 2. Unsloth Fine-Tuning (parameters for the 0.8B worker)

Base load — **16-bit LoRA** (not QLoRA; 0.8B fits the 16GB GPU, better quality):

```python
from unsloth import FastLanguageModel
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name="unsloth/Qwen3.5-0.8B",   # or local path to the Ollama-pulled weights
    max_seq_length=2048,
    load_in_4bit=False,
    load_in_16bit=True,
    full_finetuning=False,
)
```

  | param | value | rationale |

  | `lora_alpha` | 16 | α = r (Unsloth default ratio) |
  | `target_modules` | `q_proj,k_proj,v_proj,o_proj,gate_proj,up_proj,down_proj` | all linear layers — best for a small model |
  | `lora_dropout` | 0 | Unsloth optimized fast-path |
  | `bias` | "none" | — |
