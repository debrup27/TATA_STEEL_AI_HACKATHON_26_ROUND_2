# Assumptions & Limitations

## Assumptions

1. **Simulated plant.** With no live plant feed, telemetry is generated by physics-based
   models (see [EQUIPMENT_PHYSICS.md](./EQUIPMENT_PHYSICS.md)). The architecture is
   feed-agnostic — replacing the generator with a real historian/OPC-UA source needs no
   change downstream of the telemetry table.
2. **Two factories, eight assets** spanning the major steel-plant equipment classes
   (reheating, descaling, rolling, hydraulic gauge control, pickling, cold milling,
   galvanising, air-knife coating).
3. **Single-GPU, self-hosted.** qwen3.5 and BGE run locally; no external LLM/API. A CUDA
   GPU is required (low-VRAM tier for ~6–8 GB cards).
4. **Thresholds and standards** (ISO 4406, ISO 17359, L10, Paris' law, OSHA 1910.147) are
   used as realistic references; exact OEM setpoints would be tuned per real installation.
5. **Demo authentication** (`tech_demo`) — JWT is wired but not hardened for production
   (no SSO/MFA).

## Limitations

1. **Deterministic-first by design.** Health/RUL/anomaly come from the engine; the pickled
   ML is a sanity-gated refinement, not the primary predictor. This trades peak ML accuracy
   for guaranteed-sane, explainable, never-NaN outputs — the right trade for a
   decision-support tool, but it means the ML models are deliberately not the headline.
2. **Synthetic-trained ML.** Models learn from generator data, so accuracy reflects the
   simulation, not field data. Some assets (e.g. rare-fault classes) train degenerate
   models; the sanity gate makes these harmless but they add no value until retrained on
   real data.
3. **LLM quality scales with tier.** The low-VRAM 0.8b tier produces shorter, less nuanced
   narrative than the 9b; tool-calling convergence is weaker, which is why the deterministic
   aggregator fallback exists.
4. **First-boot time.** A clean machine downloads ~6.5 GB BGE + ~7 GB Ollama weights +
   corpus and trains ML — 15–30 min before fully ready. Subsequent boots reuse volumes.
5. **RAG corpus is representative, not exhaustive** — a curated set of OEM manuals, SOPs,
   ISO/OSHA excerpts and process guides, not a full plant document library.
6. **Single-node.** No horizontal scaling/HA; one Postgres, one Redis, one Ollama. Adequate
   for a plant-scale pilot, not a multi-plant deployment as-is.

## Security posture

- No external network calls at inference time (qwen/BGE local).
- Tool calls in the agentic graph are whitelisted and audit-logged; the LLM cannot mutate
  data rows directly — only through whitelisted tools with the graph's real asset_id.
- Input/output guardrails constrain scope to maintenance.
