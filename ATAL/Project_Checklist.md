# Project ATAL (Automated Troubleshooting and Asset Lifespan)

> **Agentic AI Maintenance Wizard for Tata Steel Round 2 Hackathon**

Project ATAL is an intelligent, context-aware decision-support platform designed for heavy industrial steel manufacturing environments. It consolidates fragmented data streams into a unified AI co-pilot, driving faster equipment diagnosis, root cause analysis, and predictive lifecycle modeling.

---

## 🎯 Project Goal

To shift steel plant operations from **reactive troubleshooting** to **proactive/predictive maintenance planning**, thereby reducing costly unplanned downtime, mitigating safety hazards, and optimizing spare parts management.

---

## 🏭 Project Objectives

The system must enable the following outcomes for maintenance engineers in a steel plant:

[ ] Faster and more accurate diagnosis of equipment issues
[ ] Identification of probable root causes of failures
[ ] Prediction of equipment degradation and remaining useful life
[ ] Proactive detection of abnormalities and catastrophic failure risks
[ ] Prioritization of maintenance actions based on operational and procurement constraints
[ ] Generation of structured maintenance insights and reports
[ ] Support both **reactive troubleshooting** and **proactive maintenance planning**

## 🛠 Features and Development Checklist

### 🟥 Phase 1: Core Functional Requirements

 1. [ ] Contextual Reasoning Engine (LLM/SLM)
    * [ ] Integrate a Large Language Model (LLM) or Small Language Model (SLM) for deep technical reasoning.
    * [ ] *Extra Merit:* Create or fine-tune a domain-specific model optimized for industrial maintenance.
    * [ ] Allow use of publicly available APIs.

 2. [ ] Knowledge Integration (RAG Pipeline)
    * [ ] Ingest and index technical equipment manuals.
    * [ ] Ingest and index standard operating procedures (SOPs).
    * [ ] Index historical maintenance records.
    * [ ] Index failure reports and operational logs.

 3. [ ] Natural Language Interaction
    * [ ] Support natural language queries from maintenance engineers.
    * [ ] Build multi-turn conversation support in a context-aware manner.

 4. [ ] Explainable Recommendations
    * [ ] Ensure outputs are explainable and traceable to relevant input data, historical records, rules, or documentation wherever possible.

 5. [ ] Abnormality Detection and Failure Prediction
    * [ ] Dynamic abnormality detection module for sensor streams.
    * [ ] Early warning generation framework.
    * [ ] Failure prediction for critical equipment.

 6. [ ] Feedback-Driven Improvement
    * [ ] Capture user corrections, confirmations, or outcomes to continuously improve future recommendations and learning from historical data.

 7. [ ] Real-Time Alerting Capability
    * [ ] Generate real-time abnormal alert reports.
    * [ ] Send user-specific notifications where relevant.

### 🟨 Phase 2: Ingestion and Output Pipelines

 1. [ ] Expected Inputs

    * **Operational and Failure Inputs**
      * [ ] Equipment delay logs.
      * [ ] Fault/error messages generated by equipment or control systems.
      * [ ] Failure analysis reports.
      * [ ] Incident records.
      * [ ] Historical breakdown summaries.

    * **Condition Monitoring Inputs**
      * [ ] Sensor data summaries.
      * [ ] Abnormality or anomaly alerts.
      * [ ] Process condition indicators relevant to equipment health.

    * **Knowledge and Documentation Inputs**
      * [ ] Equipment manuals.
      * [ ] Maintenance SOPs.
      * [ ] Historical maintenance records.
      * [ ] Spare parts information — availability.
      * [ ] Spare parts information — procurement lead time.

    * **User Interaction Inputs**
      * [ ] Natural language queries from maintenance engineers.
      * [ ] Scenario-based learning or troubleshooting prompts.
      * [ ] Follow-up queries in a multi-turn conversational format.

 2. [ ] Expected Outputs

    * **Diagnostic and Predictive Outputs**
      * [ ] Probable fault diagnosis.
      * [ ] Root cause analysis of failures or abnormal conditions.
      * [ ] Prediction of remaining lifecycle / remaining useful life (RUL) of equipment or systems.
      * [ ] Early warning of potential catastrophic failures in critical equipment.
      * [ ] Detection of process-related defects contributing to equipment issues.

    * **Risk and Priority Outputs**
      * [ ] Risk level classification (low, medium, high, critical).
      * [ ] Urgency assessment for intervention.
      * [ ] Bottleneck prioritization at the plant level.
      * [ ] Prioritization based on process criticality.
      * [ ] Prioritization based on delay severity.
      * [ ] Prioritization based on spares availability.
      * [ ] Prioritization based on procurement lead time.

    * **Maintenance Recommendation Outputs**
      * [ ] Step-by-step maintenance or repair recommendations.
      * [ ] Immediate action points.
      * [ ] Optimized maintenance plan.
      * [ ] Long-term monitoring recommendations.
      * [ ] Spare procurement strategy, where applicable.

    * **Reporting Outputs**
      * [ ] Structured maintenance reports.
      * [ ] Abnormal alert reports.
      * [ ] Maintenance decision summaries for engineers and supervisors.
      * [ ] Equipment-specific digital maintenance log entries *(optional enhancement)*.
