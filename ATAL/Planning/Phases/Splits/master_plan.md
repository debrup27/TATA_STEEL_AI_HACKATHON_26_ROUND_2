## 🏗️ Phase-by-Phase Scope & Implementation Details

### 🔹 Phase 1: Data Contracts & Physics Simulation Engine (June 8 – June 9)
*Focus: Isolate the system data environment and physics dependencies entirely inside containerized runtimes.*

* **Infrastructure Strategy:** Establish clean Docker peripheral layers. Deploy a secure database node to persist moving telemetry inputs and equipment configurations, and a high-performance vector node to hold text embeddings.
* **Data Layout Engine:** Map standard data objects across 6 sequential manufacturing stages ($F1$ to $F6$) to track structural flows. Establish clear input tracking properties for our three distinct product types: **Tata Astrum (HRC)**, **Tata Steelium (Automotive CRCA)**, and **Galvano (Rust-Proof GC)**.
* **Physics Framework Simulation:** Build a dedicated execution file (`simulation.py`) to stream math-constrained process variables (such as heat fluxes, rolling speeds, or vibration frequencies). Program realistic time-lag delays between sequential factory boundaries to evaluate how upstream operational drops affect downstream capacities.

### 🔹 Phase 2: Custom Frontend Console Layout (June 11)
*Focus: Build a professional, enterprise-grade Next.js/React layout running early using a local mock router context layer.*

* **iROC Control Screen Visualization:** Construct an interactive main layout resembling an industrial control console. Build out the left zone as a network tracking map to visually display resource allocation and production statuses across the 4 parallel production chains.
* **Anomalous Injector Interface:** Build a custom sidebar control dashboard containing interactive toggle buttons and input parameter sliders. This allows users to manually overwrite variables and broadcast specific equipment failures into the system state.
* **Wizard Terminal Console Component:** Implement the right zone as a real-time streaming chat layout. This component is optimized to render multi-agent thought graphs, markdown technical manual excerpts, and structured maintenance approval forms with visual state feedback.

### 🔹 Phase 3: RAG, Agent Tools Binding & Domain SLM Tuning (June 12 – June 13)
*Focus: Ingest raw operational materials, link cognitive reasoning nodes to Phase 1 backend tools, execute comprehensive test scripts, and transition your custom frontend out of mock state data.*

* **RAG Pipe Ingestion & Tool Mapping:** Parse and index unstructured engineering manuals, warehouse inventory stocks, and safety standard operating procedures (SOPs) straight into the Vector Store. Bind your multi-agent architecture execution paths directly to functional Python code hooks.
* **Frontend Socket Integration:** Discard the temporary mock routers from Phase 2. Map direct backend API and WebSocket endpoints to pipe live-updating telemetry streams and active agent reasoning tokens directly to the user dashboard components.
* **Intensive Test Automation & SLM Tuning:** Program an automated terminal test harness (`test_scenarios.py`) to systematically trigger and assert text blocks for 5 highly complex, sequential equipment failures across the plant. Apply optimization rules and strict structural system constraints on a localized Small Language Model (SLM) to guarantee secure on-prem processing without external API reliance.

### 🔹 Phase 4: Presentation, Documentation & Hand-off Packaging (June 14 – June 15)
*Focus: Zero coding. Freeze all source repositories and dedicate your complete focus to framing a winning video delivery and thorough technical documentation.*

* **Technical Video Compilation:** Record a high-quality, voiceover walk-through detailing your structural system split between Process Layers and Cognitive Layers. Capture clear scenario playthroughs showing how the Maintenance Wizard captures an early alert, reviews technical files via RAG, checks parts availability, and outputs a complete, pre-filled work order.
* **System Documentation Packaging:** Finalize a clean architectural index detailing data schemas, agent communication graphs, and reproducible execution configurations. Bundle all prototype folders, database initializations, and documentation assets into your final, production-ready submission ZIP file.
